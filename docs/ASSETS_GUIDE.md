# アプリアイコンとアセット作成ガイド

App Storeに申請するために必要なアイコンとスクリーンショットの作成方法を説明します。

## 📱 必要なアセット一覧

### 1. アプリアイコン
- **サイズ**: 1024 x 1024 px
- **フォーマット**: PNG（透過なし）
- **ファイル名**: `icon.png`
- **配置場所**: `./assets/icon.png`

### 2. スプラッシュスクリーン
- **サイズ**: 1284 x 2778 px
- **フォーマット**: PNG
- **ファイル名**: `splash-icon.png`
- **配置場所**: `./assets/splash-icon.png`

### 4. スクリーンショット
- **iPhone 6.7インチ**: 1290 x 2796 px（3-10枚）
- **iPhone 6.5インチ**: 1242 x 2688 px（3-10枚）
- **iPad Pro 12.9インチ**: 2048 x 2732 px（オプション）

### デザインのガイドライン

✅ **推奨**
- シンプルで認識しやすい
- 小さいサイズでも見やすい
- ブランドカラー（#007AFF）を使用
- 角は丸くしない（iOSが自動で丸くする）

❌ **避けるべき**
- 細かすぎるディテール
- 読みにくいテキスト
- 透過背景（iOSアイコンは透過不可）
- 他アプリに似たデザイン

## 📸 スクリーンショットの撮り方

### 方法1: iOS Simulatorで撮影

1. **Xcodeをインストール**（Macのみ）
   ```bash
   # Mac App Storeからダウンロード
   ```

2. **Simulatorを起動**
   ```bash
   open -a Simulator
   ```

3. **デバイスを選択**
   - iPhone 15 Pro Max（6.7インチ）
   - iPhone 15 Plus（6.5インチ）

4. **アプリを起動**
   ```bash
   npm run ios
   ```

5. **スクリーンショットを撮影**
   - `Cmd + S` でスクリーンショット保存
   - デスクトップに保存される

### 方法2: 実機で撮影（推奨）

1. **実機でExpo Goを起動**

2. **スクリーンショットを撮影**
   - iPhoneの音量上ボタン + サイドボタン

3. **画像を転送**
   - AirDrop、メール、またはケーブルでPCに転送

4. **リサイズ**（必要に応じて）
   - オンラインツール: https://www.iloveimg.com/resize-image

### 方法3: オンラインツールで作成

**Previewed**
- https://previewed.app/
- スクリーンショットを美しくフレーム化

**MockUPhone**
- https://mockuphone.com/
- デバイスフレーム付きスクリーンショット作成

### 撮影すべき画面

#### 必須（3-5枚）

1. **メインカレンダー画面**
   - 複数の習慣がチェックされた状態
   - カレンダーにドットが表示されている
   - 今月の達成率が表示されている

2. **個別習慣カレンダー**
   - 習慣名と統計情報（合計日数、連続記録）
   - カレンダーにチェック済みの日付が表示

3. **習慣管理画面**
   - 複数の習慣が登録された状態
   - カラフルな習慣アイコン

#### オプション（2-5枚）

4. **習慣追加モーダル**
   - 新しい習慣を入力している状態

5. **日付選択状態**
   - カレンダーで日付を選択
   - その日の習慣リストが表示

## 🖼️ アセットの配置

### 現在のファイル構造
```
DailyTrack/
├── assets/
│   ├── icon.png              # 1024x1024（要置き換え）
│   ├── splash-icon.png       # 1284x2778（要置き換え）
│   ├── adaptive-icon.png     # 1024x1024（要置き換え）
│   └── favicon.png           # 48x48（Web用、オプション）
```

### アイコンの置き換え手順

1. **新しいアイコンを作成**
   - デザインツールで作成
   - または、オンラインジェネレーターを使用

2. **ファイルを置き換え**
   ```bash
   # 既存のファイルをバックアップ
   mv assets/icon.png assets/icon.png.backup

   # 新しいアイコンをコピー
   cp /path/to/your/new-icon.png assets/icon.png
   ```

3. **サイズを確認**
   ```bash
   # Linuxで確認
   file assets/icon.png
   ```

4. **Expoを再起動**
   ```bash
   npm start
   ```

## 📊 スクリーンショットのベストプラクティス

### 構成

1. **最初の1-2枚**: 最も重要な機能
2. **中間の2-3枚**: 主要な機能
3. **最後の1-2枚**: 追加機能や特徴

### デザインのコツ

- **明るい背景**: 見やすい
- **テキストを追加**: 機能を説明
- **実際のデータ**: 空の画面は避ける
- **統一感**: 同じスタイル・色を使用

### テキストの追加

**推奨ツール**
- Canva: https://www.canva.com/
- Figma: https://www.figma.com/
- Photoshop（有料）

**テキストの内容**
- 短く（5-8語）
- 分かりやすく
- 機能の利点を説明
