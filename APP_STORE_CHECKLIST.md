# App Store 審査チェックリスト

このチェックリストを使用して、App Storeへの申請前にすべての要件が満たされていることを確認してください。

## ✅ 必須項目

### 1. アプリの基本情報
- [x] アプリ名: DailyTrack
- [x] バンドルID: com.dailytrack.app
- [x] バージョン: 1.0.0
- [x] ビルド番号: 1
- [x] アプリの説明文（日本語・英語）

### 2. プライバシーとセキュリティ
- [x] プライバシーポリシー作成済み（PRIVACY_POLICY.md）
- [x] データ収集なしの明記
- [x] NSUserTrackingUsageDescription設定
- [x] ITSAppUsesNonExemptEncryption = false 設定
- [ ] プライバシーポリシーをWebに公開（GitHub Pages、自分のサイトなど）
- [ ] App Store Connectでプライバシー情報を入力

### 3. アプリアイコンとスクリーンショット
- [ ] アプリアイコン（1024x1024 PNG）
- [ ] iPhone 6.7インチ スクリーンショット（1290 x 2796）x 3-10枚
- [ ] iPhone 6.5インチ スクリーンショット（1242 x 2688）x 3-10枚
- [ ] iPad Pro 12.9インチ スクリーンショット（2048 x 2732）x 3-10枚（オプション）
- [ ] スプラッシュスクリーン画像

### 4. アプリの機能
- [x] 基本機能が動作する
- [x] クラッシュしない
- [x] オフラインで動作する
- [x] ユーザーデータがデバイスに保存される
- [x] データ削除機能（アンインストールで削除）

### 5. App Store Connect 設定
- [ ] Apple Developer Programに登録
- [ ] App Store Connectでアプリを作成
- [ ] カテゴリ: 仕事効率化 / Productivity
- [ ] コンテンツレーティング: 4+（全年齢対象）
- [ ] 価格: 無料
- [ ] 配信可能国: すべての国（または選択）

### 6. メタデータ
- [ ] アプリ名（30文字以内）
- [ ] サブタイトル（30文字以内）
- [ ] プロモーションテキスト（170文字以内）
- [ ] 説明（4000文字以内）
- [ ] キーワード（100文字以内）
- [ ] サポートURL（任意）
- [ ] マーケティングURL（任意）

### 7. ビルドとテスト
- [ ] EASでproductionビルド作成
- [ ] TestFlightでテスト
- [ ] すべての機能が正常に動作することを確認
- [ ] 様々なデバイスサイズでテスト

## 📄 審査用情報

### デモアカウント情報
- **不要**（ログイン機能なし）

### 審査ノート（Review Notes）
```
このアプリは習慣トラッキングアプリです。
以下の機能を提供します：

1. 習慣の追加・削除
2. 日付を選択して習慣をチェック
3. カレンダーで達成状況を可視化
4. 個別習慣の統計情報表示

すべてのデータはローカルに保存され、インターネット接続は不要です。
個人情報の収集は一切行いません。

テスト手順：
1. 「管理」タブから習慣を追加
2. 「全習慣」タブで日付を選択して習慣をチェック
3. 各習慣のタブで個別カレンダーを確認
```

## ⚠️ よくある審査却下理由と対策

### 1. 最小限の機能要件
**問題**: アプリの機能が不十分
**対策**: ✅ 完全な習慣トラッキング機能を実装済み

### 2. プライバシーポリシー不足
**問題**: プライバシーポリシーが不明確
**対策**: ✅ PRIVACY_POLICY.md作成済み（Webに公開が必要）

### 3. クラッシュ・バグ
**問題**: アプリがクラッシュする
**対策**: ✅ エラーハンドリング実装済み

### 4. UI/UXの問題
**問題**: UIが不完全または分かりにくい
**対策**: ✅ 直感的なUI設計済み

### 5. メタデータの不一致
**問題**: スクリーンショットと実際の機能が異なる
**対策**: 実際の画面をスクリーンショットとして使用

- [ ] Production モードでビルド

- [ ] ビルド番号がインクリメントされている

## 📋 提出手順

1. **EASでビルド**
   ```bash
   eas build --platform ios --profile production
   ```

2. **TestFlightにアップロード**
   ```bash
   eas submit --platform ios
   ```

3. **TestFlightでテスト**
   - 内部テスターに配布
   - すべての機能をテスト

4. **App Store Connectで設定**
   - メタデータ入力
   - スクリーンショットアップロード
   - プライバシー情報入力

5. **審査に提出**
   - 「審査に送信」ボタンをクリック
   - 審査ノートを入力

6. **審査結果を待つ**
   - 通常1-3日
   - 初回は時間がかかる場合あり
